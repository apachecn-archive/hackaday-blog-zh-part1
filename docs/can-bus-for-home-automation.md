# 用于家庭自动化的 CAN 总线

> 原文：<https://hackaday.com/2012/03/07/can-bus-for-home-automation/>

这是新的[家庭自动化系统](http://blackrynius.wordpress.com/2012/02/29/k-node/)上的一个节点，Black Rynius 正在开发。到目前为止，他只测试了两个节点的系统，但计划随着项目的进展建立更多的节点。他选择使用 CAN 总线进行通信；汽车应用中最常见的协议。

使用 CAN 总线的最大好处是它只需要一对通信线。正如你所看到的，在这个板上有一个旧的门铃，他希望使用现有的门铃电线来连接节点。每个单元包括一个 PIC 18F4580，它内置了一个 CAN 引擎，便于协议转换。还有一个处理传输的 MCP2551。你可以在他的实验贴中读到更多关于硬件选择的信息。

到目前为止，几乎一切都在按计划进行。他能够在两块电路板之间发送和接收数据，包括来自传感器的温度和来自 DS1305 RTC 芯片的时间。令他烦恼的一件事是那个门铃。它为给电路板供电的壁式电源消耗了太多的电流，使微控制器断电并导致复位。这不是一个很难解决的问题，我们期待在不久的将来有更多的发展。
