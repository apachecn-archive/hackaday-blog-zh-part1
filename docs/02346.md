# 黑帽 2008: Pwnie 颁奖典礼

> 原文:[https://hack aday . com/2008/08/06/black-hat-2008-pwnie-award-ceremony/](https://hackaday.com/2008/08/06/black-hat-2008-pwnie-award-ceremony/)

![](../Images/206b3a6d5b7f0e282346c57d60529915.png)
[黑帽](http://mahalo.com/Black_Hat)发布会第一夜随着 [Pwnie 颁奖典礼](http://pwnie-awards.org/2008/)落幕。该奖项奖励在安全方面的成就…但大多是失败。值得注意的是，这是第一年有人亲自领奖。Hack a Day 通过生产 MacBook mini-DVI 到 VGA 适配器(如上图)取得了早期的胜利。仪式在那之后相当直接。[最佳服务器端漏洞](http://pwnie-awards.org/2008/awards.html#bestserverbug)去了 [Windows IGMP 内核漏洞](http://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2007-0069)。这是默认 Windows 防火墙中的远程内核代码执行漏洞。最佳客户端漏洞[出现在多个 URL 协议处理缺陷上，比如这个](http://pwnie-awards.org/2008/awards.html#bestclientbug) [URI 漏洞](http://www.hackaday.com/2007/10/21/toorcon-9-uri-use-and-abuse/)。[群发](http://pwnie-awards.org/2008/awards.html#mass0wnage)去找 [WordPress](http://www.mahalo.com/Wordpress) 很多很多漏洞。[最具创新研究](http://pwnie-awards.org/2008/awards.html#research)去了[冷启动攻击](http://www.hackaday.com/2008/07/18/hope-2008-cold-boot-attack-tools-released/)团队。McAfee 赢得了最烂的供应商回应，称 XSS 不能被用来黑服务器。最被夸大的错误是【丹·卡明斯基】的 [DNS 漏洞](http://www.hackaday.com/2008/08/06/black-hat-2008-dan-kaminsky-releases-dns-information/)。最大的失败是由 Debian 背后的团队赢得的，他们发布了 [OpenSSL bug](http://www.itnews.com.au/News/76080,openssl-bug-found-in-debian-linux.aspx) 整整两年。[终身成就奖](http://pwnie-awards.org/2008/awards.html#lifetime)由[蒂姆·纽瑟姆](http://www.google.com/search?q=%22tim+newsham%22)获得。最后，[最佳歌曲](http://pwnie-awards.org/2008/awards.html#bestsong)是由卡巴斯基实验室为 Packin ' The K！，您可以在下面找到嵌入的。
