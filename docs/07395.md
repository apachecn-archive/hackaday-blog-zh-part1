# CTRL-ALT-DELETE 的由来

> 原文:[https://hack aday . com/2011/04/15/the-origin-of-ctrl-alt-delete/](https://hackaday.com/2011/04/15/the-origin-of-ctrl-alt-delete/)

[![](../Images/ed3051b3089d25e6b10a4d07dce35cab.png "ctrl-alt-delete")T2】](http://hackaday.com/wp-content/uploads/2011/04/ctrl-alt-delete.png)

你可能从来没有想过这个问题，但是在技术历史的某个时刻，使用得太频繁的键盘组合 Control + Alt + Delete 肯定是由某个随机编码者带来的。但是等等，它不仅仅是一个随机的编码器。击键组合归功于[大卫·布莱德利]。他是 IBM 个人电脑的最初设计者之一。休息过后，你甚至可以在视频中听到他自己讲述的故事。

在他的软件测试团队每次重启都要等待开机自检(POST)例行程序完成，对此他越来越厌倦，于是他想出了这个主意。我们记得硬件缓慢的旧日时光，也能理解他在[失去时间](http://xkcd.com/303/)时的沮丧。他决定引入一种快捷方式，允许软件重启而无需重启硬件。最初的实现使用 CTRL-ALT-ESC，但后来被更改，以便一个受挫的键盘 mash 不会意外地重新启动系统。
