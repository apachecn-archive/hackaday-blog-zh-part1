</span> <p>【丹尼斯】正在用<a href="http://www.youtube.com/watch?v=CkGqn5rNzK8" target="_blank">一只机械臂作为象棋对手</a>。一只<a href="http://www.lynxmotion.com/Product.aspx?productID=672&amp;CategoryID=" target="_blank"> Lynxmotion AL5A </a>机械臂不是使用<a href="http://hackaday.com/2010/02/20/rube-goldberg-ftw/">的板下移动系统</a>，而是将每一片取出，并将其移动到下一个空间。他告诉我们，他正在使用自己创建的 Python 脚本来处理这些动作，并决定下一步该做什么。这肯定意味着他在用网络摄像头捕捉棋盘上棋子的位置。大约进行到一半的时候，你可以看到机器人撞上了一个棋子。我们想知道随着游戏的进行，当棋子离每个方块的中心越来越远时，他是否有捡起棋子的问题。从我们可以看到，看起来像一个伟大的工作！</p> </body> </html>
