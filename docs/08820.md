# 使用 ATtiny85 的 N64 控制器输入

> 原文:[https://hack aday . com/2011/10/13/n64-controller-input-using-an-attini 85/](https://hackaday.com/2011/10/13/n64-controller-input-using-an-attiny85/)

![](../Images/4e7495cddbf2d82905f0fa47c7ceb3f0.png "read-n64-controllers-using-an-ATtiny85")

[Larsim]使用 ATtiny85 微控制器计算出了从 N64 控制器读取按钮和操纵杆数据的必要时序。当他在垃圾箱里发现这对控制器时，这个项目就诞生了。我们经常截获运往垃圾填埋场的好东西，尤其是在[嬉皮士圣诞节](http://www.citydictionary.com/WI/Madison/Hippie-Christmas/3998/)，所有学生同时换公寓。

[Larsim]没有打开控制器，直接给按钮打补丁，而是查找连接器的引脚，接入串行数据线。在真正的黑客时尚中，他使用两个 5V 线性调节器和一个串联二极管将电压源降至接近 3.6V，因为他手头没有可变调节器。这听起来确实会导致噪音，如果读数错误，这可能会导致噪音，但可以通过下一个零件订单来解决。

控制器在回显嵌入了按钮数据的响应之前等待轮询信号。这一过程非常快，如果没有晶振，芯片需要配置为使用内部 PLL 将 R/C 振荡器斜坡提升至 16Mhz。现在芯片运行足够快，外部中断从控制器读取串行响应，代码根据该输入做出反应。

似乎这些 N64 控制器被扔进垃圾桶的最大原因是因为模拟操纵杆磨损了。如果你有疯狂技能[，你可以用不同类型的](http://hackaday.com/2011/02/08/replace-an-n64s-worn-out-joystick/)来代替它。
