# 窥探 Iclicker 的硬件和固件

> 原文:[https://hack aday . com/2011/10/23/snooping-around-in-the-IC licker-hardware-and-firmware/](https://hackaday.com/2011/10/23/snooping-around-in-the-iclicker-hardware-and-firmware/)

![](../Images/72403b63c538e64a31b1d75b2047578a.png "reverse-engineering-the-iclicker")

[Arko]被迫购买 iclicker 用于他的一些大学课程。它的大小与电视遥控器相似，只是它只有六个按钮，并且通过无线电频率而不是红外线进行通信。这个想法是，教室里有一个教师使用的基站，他或她可以向班级提问，并获得即时反馈。结果通常会被投影到屏幕上，让所有人都可以看到，但只有教师才能知道谁以何种方式回答了问题。在[Arko 的]案例中，职业奖励的参与点数只能通过使用该设备获得。他决定通过对设备进行逆向工程，从花费中学习一些东西。

初步的硬件检查告诉他，它使用 ATmega8 微控制器，并且有一个标准的 6 引脚 ISP 封装，只等着安装表面贴装引脚接头。一旦他焊接在那个头上，他试图读出固件，但 iClicker 自动复位。他猜测电源线和地线有问题，所以他直接焊接到它们上，并能够转储数据——安全保险丝没有设置。他继续在 EEPROM 中窥探器件 ID 的存储位置，然后观察一些 SPI 通信，看看微控制器向无线电芯片发送了什么。但是还有很多有待发现，他计划至少发表两篇后续文章来分享他的发现。
