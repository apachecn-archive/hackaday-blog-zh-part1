# 3D 打印机电计算机

> 原文：<https://hackaday.com/2011/10/27/3d-printed-electromechanical-computer/>

![](img/9874d9c16df8a98ea3b6451121b6daad.png "gear")

几天前的晚上，[克里斯·芬顿]在纽约电阻器公司闲逛，花一些时间在他的机电计算机项目上。你可能会从他的[小克雷](http://hackaday.com/2010/09/29/tiny-cray-1-courtesy-of-an-fpga/)中想起【克里斯】，他正在[的](http://hackaday.com/2011/09/08/recovering-data-for-a-homemade-cray/)上安装操作系统。看来[克里斯]要回到大约 150 年前，他已经将目光瞄准了[巴贝奇]的[分析引擎](http://en.wikipedia.org/wiki/Analytical_Engine)的 3D 打印版本。

分析引擎~~被~~是工程和机械加工的一个非凡壮举。这是第一台可编程计算机。遗憾的是，它从未在 19 世纪建成。[Chris]并没有像[Babbage]的多项式计算[差分引擎](http://en.wikipedia.org/wiki/Difference_engine)那样建造一个美化了的计算器——他正在全力以赴，建造一些有条件循环的东西。

[Chris]称他的设备为机电计算机，所以我们假设它不会像大英科学博物馆里的版本那样由曲柄驱动。现在，他已经构建了对他设计的 ALU 至关重要的十进制计数齿轮。所有的零件都被打印在 Thingomatic 上，所以我们打赌[Chris]将会严重依赖 MakerBot [自动化构建平台](http://www.makerbot.com/blog/2010/09/13/makerbot-automated-build-platform/)来制造成千上万的零件。
