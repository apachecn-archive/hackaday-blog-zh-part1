</span> <p>[Cal Henderson]在第一届<a href="http://djangocon.org/" target="_blank"> DjangoCon </a>年会上发表了题为<em>我为什么讨厌 Django </em>的主题演讲。<a href="http://www.djangoproject.com/" target="_blank"> Django </a>是一个开源的 BSD 许可的 web 框架，用 Python 编写。谷歌已经将主题演讲的全部内容发布到 YouTube 上，你可以在上面找到嵌入的内容。虽然这个演讲很幽默(并对 Rails 开发人员进行了许多抨击)，但它确实提供了关于什么是好的 web 框架的见解。[Cal]是 Flickr 的工程总监，也是如何扩大网站规模的权威。他指出，大多数框架旨在让项目快速起步，但在构建更大的服务时却缺乏这种能力。他谈到了 Django 的一些需要改进的地方。这真的是一个有趣的视角，看看要做大需要什么。<span id="more-3276"/></p> <p>在结尾，[Cal]提到 Django 没有吉祥物，“我可以认真对待一个有着神奇力量的吉祥物的框架。”嗯，这似乎是最容易解决的事情，所以 avalonstar 创建了<a href="http://avalonstar.com/blog/2008/sep/9/web-framework-ponies/" target="_blank"> Django:具有神奇力量的小马的网络框架</a>。它可以作为壁纸出现在<a href="http://www.djangopony.com/" target="_blank"> djangopony </a>上，而 Ruby legend【为什么】，已经着手<a href="http://hackety.org/2008/09/15/documentsRevealDjangoPonyTailOfLies.html" target="_blank">揭穿其中的魔法</a>。</p> </body> </html>
