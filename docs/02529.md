# 为什么我讨厌姜戈

> 原文：<https://hackaday.com/2008/09/16/why-i-hate-django/>

[https://www.youtube.com/embed/i6Fr65PFqfk?version=3&rel=1&showsearch=0&showinfo=1&iv_load_policy=1&fs=1&hl=en-US&autohide=2&wmode=transparent](https://www.youtube.com/embed/i6Fr65PFqfk?version=3&rel=1&showsearch=0&showinfo=1&iv_load_policy=1&fs=1&hl=en-US&autohide=2&wmode=transparent)

[Cal Henderson]在第一届 [DjangoCon](http://djangocon.org/) 年会上发表了题为*我为什么讨厌 Django* 的主题演讲。 [Django](http://www.djangoproject.com/) 是一个开源的 BSD 许可的 web 框架，用 Python 编写。谷歌已经将主题演讲的全部内容发布到 YouTube 上，你可以在上面找到嵌入的内容。虽然这个演讲很幽默(并对 Rails 开发人员进行了许多抨击)，但它确实提供了关于什么是好的 web 框架的见解。[Cal]是 Flickr 的工程总监，也是如何扩大网站规模的权威。他指出，大多数框架旨在让项目快速起步，但在构建更大的服务时却缺乏这种能力。他谈到了 Django 的一些需要改进的地方。这真的是一个有趣的视角，看看要做大需要什么。

在结尾，[Cal]提到 Django 没有吉祥物，“我可以认真对待一个有着神奇力量的吉祥物的框架。”嗯，这似乎是最容易解决的事情，所以 avalonstar 创建了 [Django:具有神奇力量的小马的网络框架](http://avalonstar.com/blog/2008/sep/9/web-framework-ponies/)。它可以作为壁纸出现在 [djangopony](http://www.djangopony.com/) 上，而 Ruby legend【为什么】，已经着手[揭穿其中的魔法](http://hackety.org/2008/09/15/documentsRevealDjangoPonyTailOfLies.html)。
