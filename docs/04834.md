不久前[我们问我们的读者](http://hackaday.com/2010/01/09/ask-a-winner/)他们想从推送 N900 的赢家和他们的黑客那里听到什么。我们得到了一些愚蠢的问题，一些严重的，我们问了两个，现在推团队已经回答了。

触觉指导团队今天为我们花了一点时间。他们的 N900 hack 是一条皮带，可以通过电机帮助你指向正确的方向。但是在我们采访之后，我们发现他们有更大的计划。记得查看[他们的博客](http://blogs.nokia.com/pushn900/category/the-haptic-guide/)更新——包括[关于他们进展的新视频](http://www.youtube.com/watch?v=hAd2n-6ZFPk)(可爱的柔性电路板家伙们！)

>感谢您抽出时间回答 HackaDay 的一些问题。我们喜欢黑客愿意与我们分享他们的工作。

>你和你的团队是怎么走到一起的？有什么有趣的背景故事想分享吗？

[Mike]和[Tim]在戴尔共事了将近两年。埃德蒙顿的呼叫中心关闭后，他们保持联系，并很快决定在 2008/2009 年的微软“嵌入式公园”竞赛中相互竞争。尽管他们相互竞争，但他们经常在一起研究 Window CE 开发工具。[Tim]擅长软件开发，而[Mike]擅长硬件开发。在互相帮助完成他们的项目时，他们发现他们组成了一个非常好的团队。当诺基亚推比赛开始时，他们决定一起努力。

>你和你的团队合作过其他项目吗？蜂窝相关项目？N900 项目？

我们参加了微软“嵌入式公园”2008/2009 竞赛，并且都进入了 2009/2010 竞赛的第二轮。作为业余爱好者，我们偶尔也会单独或一起做兼职项目。
PUSH N900 竞赛是我们进入手机黑客世界的第一个合资项目。[Tim]之前曾尝试过为 Windows Mobile 设备构建应用程序，但没有尝试过其他平台。

>你团队项目的主要灵感是什么？

[Mike]自从他第一次读到触觉罗盘，他就喜欢上了这个想法。我们花了一个晚上的时间进行头脑风暴，为推动比赛做准备，用触觉罗盘做一些事情的想法不断浮出水面。我们决定扩展这个想法，因为触觉罗盘本身并不新鲜。相反，我们认为它应该与 N900 这样的手机完美搭配，赋予它更多的功能，甚至是社交功能。

>你认为当你完成/发布时，人们会有什么反应？

我们很大程度上怀疑人们会有两种反应。首先，会有一大群人从未见过类似的产品，他们会以开放的心态看待它，并看到该产品的巨大潜力。第二，会有一群人以前见过触觉罗盘，会错误地认为这个项目只是一个非原创的复制品。我们希望第二组能重新审视这个项目，并意识到腰带本身只是一个帮助实现更大图景的工具。触觉向导项目的前景比大多数人想象的要广阔得多。博客的主要焦点一直围绕着腰带，这是一个关键的组成部分，但事实是 N900 赋予了这个项目它的主要价值。N900 能够拍摄照片，并自动用地理标签进行编码。这些照片可以在网上分享或发送给朋友，他们可以使用 N900 和触觉向导软件加载照片，并以一步一步的方式自动引导到照片拍摄的位置。这提供了很大的价值，因为您不再需要键入或输入您想要去的位置，无论是通过地址还是通过纬度/经度坐标。你只需要一张位置的照片，该软件就像你在商业 GPS 导航器上看到的那样，将当前位置和目标位置依次分解。这就是皮带发挥作用的地方；它消除了不断看屏幕或听方向的需要。现在，你可以把手机放在支架上或口袋里，走在街上，下意识地跟随皮带的振动，而不受周围环境的干扰。这是完美的旅游设备。

>你对你的团队有什么未来的计划吗？继续 N900 开发？更新您当前的项目？

展望未来，我们计划开发类似腰带设计的其他设备，这样可穿戴设备的选择就不止一种了。这些东西可能包括腕带或臂章、带灯指示方向的手表、帽子等。此外，我们可能会考虑建立一个在线地理编码照片共享服务的可能性，这样你就可以计划游览你要去的地方。拍下地标的照片，发布到服务上，该地区的人们可以找到照片，并前往该地点。

>你对其他获奖者有什么看法？

比赛的其他获胜者都很棒。团队中有非常有趣的人物组合，从极客到艺术家。此外，这些团队在分享信息方面做得很好。例如，当[Tim]开始应用程序开发时，Lighthack 团队的 Introcept 很乐意分享他的蓝牙实现细节，这给了[Tim]他所需要的东西。

>我们评论员的提问。

>你对 Arduino 有什么想法？您打算在您的项目中使用它吗？

不可否认，Arduino 是一个很好的学习和原型工具。对于我们的原型，我们使用 Arduino Mini Pro。我们选择这个是因为我们可能需要的尺寸限制、输出和处理能力。在生产环境中，我们会使用合适的芯片并直接设计电路。但最终，这只是一个原型，Arduino 非常适合它。

>N900 的硬件和软件使用情况如何？

起初，使用 N900 就像跳进游泳池的深水区，却不知道如何游泳。但这主要是因为[Tim]以前从未使用过 N900/Linux 开发工具。任何熟悉 Linux 环境中的 C、C++或 Python 编码的人在为其创建应用程序时都不会有任何问题。开发工具非常兼容。在许多情况下，要让一个桌面 GTK 应用程序在手机上工作，最基本的就是使用 Scratchbox 编译器重新编译源代码，并将文件复制到手机上。

>你如何看待手机转向开源软件，比如 Maemo？你打算或者曾经尝试过 Android 吗？

开源软件几乎总是朝着正确的方向迈出的一步。Maemo 环境的真正价值在于，开源工具允许您简单地复制一个 Python 脚本，例如，从一个平台复制到另一个平台，它就能工作。[Tim]已经考虑未来的 Android 开发，但由于其开放平台，可能会坚持 N900。

>(如果您使用辅助电源，例如运行电机)您的项目预计电池续航时间是多少？

我们目前正在使用一个 2000 毫安的锂聚合物电池为触觉带供电。由于我们选择使用的电机，皮带的电池寿命相当长:持续使用约 12 小时。

>我们能得到一份你在项目中使用的物品清单吗？(示例–蓝牙设备、电机、专用传感器等)

对于我们的原型，我们选择使用大多数标准的模块化部件，其中大部分可以在 sparkfun.com 购买。每条皮带有以下几个主要部件:
1 条 Arduino Pro Mini (3.3V)
1 条 BlueSmirf 黄金模块
1 条 HMC6352 指南针模块
8 条振动电机
1 块聚合物锂离子电池——2000 mAh
一条双层皮带
一条诺基亚 N900(可选。没有它，腰带就像一个普通的指南针，总是指向磁北极)
BlinkM I2C 发光二极管(可选)
