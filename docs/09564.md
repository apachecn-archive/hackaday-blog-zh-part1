# 覆盆子 Pi 跑 XBMC；可靠地解码 1080p

> 原文:[https://hackaday . com/2012/01/24/覆盆子-pi-runs-xbmc-可靠解码-1080p/](https://hackaday.com/2012/01/24/raspberry-pi-runs-xbmc-reliably-decodes-1080p/)

![](../Images/dd144c48944509cfe4c8606be0275486.png "raspberry-pi-running-xbmc")

这是 Raspberry Pi 板，基于 ARM 的 GNU-Linux 计算机。我们对此有所耳闻，但最近当这台机器以 1080p 的速度运行 XBMC 时，它引起了我们的注意。小封装需要完成大量解码工作，这是在硬件层面上完成的。

普通读者会知道我们是 XBMC 项目的粉丝，并且一直在寻找一种可以贴在电视机背面的小型设备。我们曾希望它会是比格犬，但那从未真正实现。但这看起来真的很有潜力，并且价格标签为 35 美元(这是为了更大的 256MB RAM 选项)，这是显而易见的。

现在仍然有很多传言。我们遇到了一个推测该设备不能很好地解码除 h.264 以外的视频格式的线程，因为它只对该编解码器使用硬件解码。我们将保留判断，直到有更可靠的信息。但是你可以通过[这个论坛主题](http://forum.xbmc.org/showthread.php?t=113824&page=12)来挖掘，在这里，一直从事硬件工作的 XMBC 开发人员参与了讨论。
