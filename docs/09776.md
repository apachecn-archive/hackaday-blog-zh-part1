# 可视化一纳秒

> 原文:[https://hackaday.com/2012/02/27/visualizing-a-nanosecond/](https://hackaday.com/2012/02/27/visualizing-a-nanosecond/)

我们很高兴看到这个视频，在这里[少将·格蕾丝·赫柏] [解释了如何可视化纳秒](http://www.youtube.com/watch?v=JEpsKnWZrJ8)。现在我们以前从未听说过[格蕾丝·赫柏]，但是一旦你看了这个剪辑(也是在休息后嵌入的)，你会想知道这个人是谁。当用微控制器开发时，我们总是以秒为单位工作。但是这些概念太抽象了，我们从来没有必要把它们看作是物理距离。毕竟它们是时间的量度，对吗？

你看不清，但她手里拿着一根电线。它有 11.8 英寸长，代表电在一纳秒(一秒的十亿分之一)内可以传播多远。她继续解释说，这是对光在一纳秒内可以传播的距离的计算，然后当她用它来解释卫星通信中的延迟时，真正击中了这个概念。对我们来说，当芯片卡在循环中等待中断时，不将芯片置于睡眠模式的浪费就是我们建立连接的地方。

所以回到这个女人身上。我们认为你会非常喜欢阅读她的维基百科传记页面。[Grace]是计算机科学的先驱。人们认为她编写了第一部计算机编译器。她假设并阐明了导致 COBOL 发展的概念，并推广了术语“调试”。简而言之，她是我们都站在肩上的巨人之一。
