# 开车，不去任何地方

> 原文：<https://hackaday.com/2010/04/23/driving-the-car-without-going-anywhere/>

 <https://www.youtube.com/embed/psAZSPK4pWc?version=3&rel=1&showsearch=0&showinfo=1&iv_load_policy=1&fs=1&hl=en-US&autohide=2&wmode=transparent>

</span> <p>这个<a href="http://nippe.teknopapu.net/index.php/archives/57#more-57" target="_blank">视频游戏控制器</a>是一个工厂新鲜的大众。很像本周早些时候的<a href="http://hackaday.com/2010/04/18/racing-sim-with-real-car-parts/">赛车模拟器</a>，视频游戏数据显示在仪表板上。这将我们带到了一个更高的层次，因为游戏的控制是通过汽车的<a href="http://en.wikipedia.org/wiki/Canbus" target="_blank"> CANbus </a>和<a href="http://en.wikipedia.org/wiki/On-board_diagnostics#OBD-II_Signal_Protocols" target="_blank"> ODB-II </a>连接器实现的。如果你不说汽车术语，这意味着来自方向盘、换挡杆和踏板的传感器读数被拾取，并作为操纵杆命令输出到运行驾驶游戏的 PC。体验中唯一使用真实事物替代品的地方是声音，它是通过扬声器播放的，而不是从引擎盖下发出的。看起来你只需要给你的车库添加一个投影仪和屏幕，就可以把它变成最热门的新游戏设备。</p> </body> </html>
